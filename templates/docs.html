System Overview

SheetBase is a hybrid data management system designed specifically for electronic components. It bridges the gap between static PDF datasheets and structured SQL databases using AI-driven classification and extraction.

The primary goal is to automate the tedious process of manual parameter entry, allowing engineers to simply upload a datasheet and have the system intelligently route the data to the correct inventory table.

Smart Upload & AI Routing

The core innovation of SheetBase is the AI Router. Unlike traditional systems that require you to specify a category before uploading, SheetBase analyzes the PDF content in real-time.

1. Classification

The AI identifies the component type (OPV, BJT, or MOSFET) based on header information and parameter patterns.

2. Parameter Extraction

Technical parameters are extracted and mapped to the specific SQL schema for that component type.

3. Routing

The data is inserted into the correct table independently of the user's current dashboard view.

4. Validation

The system provides a toast notification confirming the routing status and any extracted metadata.

API Reference

GET /data/<category>

Fetches all records for a specific category (opv, bjt, or mosfet).

POST /upload-pdf

Handles multipart PDF uploads for AI-driven extraction and routing.

GET /test-db

Verifies the health of the connection to the remote PostgreSQL database.

Terminal Testing Tools

For developers, we provide standalone scripts to test database interactions without using the web interface.

test-post.py

Inserts sample components (OPA2134, 2N2222A, IRFZ44N) with full parameters into the DB.

python3 test-post.py

test-conn.py

Performs a low-level connectivity check and basic read/write tests.

python3 test-conn.py
